## Цель

Реализовать карточку юр. лица как на референсе и аккуратно встроить новый блок «Банкротство»: краткая сводка в карточке + подробная хронология в боковой панели.

## Структура страницы

Маршрут `src/routes/index.tsx` — поиск + карточка компании. Тестовые данные — локальный mock-объект (ПАО СБЕРБАНК), без API.

## Блок «Банкротство» в карточке (свёрнутый вид)

Одна компактная секция с разделителем сверху:
- Бейдж статуса дела (цветной): «Наблюдение» / «Финансовое оздоровление» / «Внешнее управление» / «Конкурсное производство» / «Завершено» / «Прекращено».
- Номер дела + суд (короткой строкой).
- Арбитражный управляющий: ФИО, СРО (одна строка, с иконкой).
- Последнее событие: дата + краткое название.
- Кнопка «Открыть хронологию» → открывает Sheet.

## Боковая панель (Sheet) — подробная хронология

Используется shadcn `Sheet` (справа, ширина ~520px, скролл внутри).

Содержимое:
1. Заголовок: «Банкротное дело» + бейдж текущего статуса.
2. Сводка: номер дела, суд, дата возбуждения, текущая стадия, дата начала стадии.
3. Арбитражный управляющий: ФИО, СРО, контакты (если есть).
4. Кредиторы — короткая сводка (сумма требований, кол-во, реестр) — опционально, числа из mock.
5. Таймлайн событий (вертикальный):
   - дата
   - тип события (Заявление о банкротстве, Введение наблюдения, Утверждение АУ, Собрание кредиторов, Введение конкурсного производства, Включение требований, Реализация имущества, Завершение и т.п.)
   - краткое описание
   - источник/документ (текст)
   События отсортированы по дате (новые сверху), визуально сгруппированы по стадиям.

## Модель данных (mock, TypeScript)

Файл `src/lib/bankruptcy-mock.ts`:

```ts
export type BankruptcyStage =
  | 'none' | 'monitoring' | 'financial_recovery'
  | 'external_management' | 'bankruptcy_proceedings'
  | 'completed' | 'terminated';

export interface BankruptcyEvent {
  id: string;
  date: string;        // ISO
  stage: BankruptcyStage;
  title: string;
  description?: string;
  document?: string;
}

export interface BankruptcyCase {
  status: BankruptcyStage;
  caseNumber: string;       // А40-...
  court: string;
  filedAt: string;          // дата возбуждения
  stageStartedAt: string;
  manager: { fullName: string; sro: string; contact?: string };
  creditors?: { count: number; totalClaims: number; registryClosedAt?: string };
  events: BankruptcyEvent[];
}
```

Карточка компании получает `bankruptcy?: BankruptcyCase` (необязательное).

Используем уже доступные shadcn-компоненты: `card`, `button`, `badge`, `sheet`, `separator`, `scroll-area`, иконки lucide-react. Стили — через семантические токены из `src/styles.css`, без хардкода цветов; для статусов банкротства добавим маппинг вариантов badge (variant + tailwind-классы на основе токенов).

## Технические детали

- Состояние открытия Sheet — локальный `useState` в `CompanyCard`.
- Таймлайн — простая вёрстка: левая колонка с маркером и линией (`bg-border`), правая — карточка события.
- Даты форматируем через `Intl.DateTimeFormat('ru-RU')`.
- Доступность: у кнопки «Открыть хронологию» — понятный aria-label; в Sheet корректный `SheetTitle`/`SheetDescription`.
- SEO: в `Route.head` для index — title «Поиск юридической информации», описание.
